
# Contributing

## Style & quality

- Prefer small PRs with focused changes.
- Add docstrings/JSDoc for all exported/public code.
- Write tests for pure logic where possible.

## Python

- Use type hints.
- Prefer `ruff` for linting.
- Avoid heavy work at import-time.

## TypeScript

- Keep components focused.
- Document exported functions/components with JSDoc.
